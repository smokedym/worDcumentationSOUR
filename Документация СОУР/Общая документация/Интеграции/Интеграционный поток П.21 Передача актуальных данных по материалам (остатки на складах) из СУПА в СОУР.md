## Параметры потока

| Параметр      | Значение                                                                                  |
| :------------ | :---------------------------------------------------------------------------------------- |
| Направление   | СУПА (SAP) → СОУР                                                                         |
| Протокол      | SOAP/XML                                                                                  |
| Транспорт     | ЕИП (Единая интеграционная платформа)                                                     |
| Триггер       | Периодический опрос таблицы `materials_on_storage_update`                                 |
| Периодичность | Ежеминутно для складов, по которым зафиксировано изменение (наличие ID в таблице-очереди) |
| Обработка     | Запись в таблицу `material_on_storage`                                                    |
| Маршрутизация | По HTTP-заголовку `header_sourcePlatform` (center или cip)                                |
|               |                                                                                           |

## Общая информация

- **Назначение:** Синхронизация данных об остатках МТР на складах в режиме реального времени для обеспечения корректного списания материалов при закрытии заказов ТОРО в СОУР.
- **Инициатор:** Интеграционная шина (ЕИР).
- **Процесс:** Система ежеминутно проверяет таблицу `materials_on_storage_update`. Для каждого найденного `storage_id` формируется SOAP-запрос к мастер-системе СУПА для получения текущего перечня материалов и их количества.

## Запрос (СОУР → СУПА)

| Тег (Центр) | Тег (ЦиП) | Описание | Источник данных в СОУР |
| :--- | :--- | :--- | :--- |
| `ZPM_SOUR_MATERIALS` | `urn:ZpmSourMaterial` | Наименование операции | — |
| `IV_PLANT` | `Werks` (в `<item>`) | Код завода | `storages.integration_entity_id` |
| `IV_WAREHOUSE` | `Lgort` (в `<item>`) | Код склада | `storages.integration_entity_code` |

## Ответ (СУПА → СОУР)

| Тег (Центр) | Тег (ЦиП) | Поле БД СОУР | Описание |
| :--- | :--- | :--- | :--- |
| `PLANT` | `Werks` | — | Код завода (используется для поиска склада) |
| `WAREHOUSE` | `Lgort` | `storage_id` | Код склада (FK к `storages`) |
| `MATERIAL` | `Matnr` | `material_id` | Номенклатурный номер (FK к `materials`) |
| `CHARG` | `Charg` | `batch_number` | Номер партии (если есть) |
| `QUANTITY` | `Labst` | `count` | Актуальное количество на складе |

## Ключевые правила обработки

- **Идентификация объектов:** Поиск склада и материала в соответствующих справочниках СОУР осуществляется по паре `integration_entity_id` + `source_platform`.
- **Обновление остатков:** При получении данных по конкретному складу система полностью удаляет старые записи об остатках для этого склада в `material_on_storage` и записывает новые.
- **Партионный учет:** Если СУПА передает данные в разрезе партий (`CHARG`/`Charg`), остатки записываются отдельными строками для каждой партии.
- **Обработка сбоев:** При получении ошибки (например, HTTP 404) или некорректных данных, ЕИР обеспечивает сохранение сообщения в очереди и инициирует механизм Retry (повтор отправки).

## Примеры

### Пример запроса (Центр)

```xml
<soapenv:Body>
  <ZPM_SOUR_MATERIALS>
    <IV_WAREHOUSE>305R</IV_WAREHOUSE>
    <IV_PLANT>3100</IV_PLANT>
  </ZPM_SOUR_MATERIALS>
</soapenv:Body>
```

### Пример ответа (Центр)

```xml
<soap-env:Envelope xmlns:soap-env="http://schemas.xmlsoap.org/soap/envelope/">
  <soap-env:Header/>
  <soap-env:Body>
    <n0:ZPM_SOUR_MATERIALSResponse xmlns:n0="urn:sap-com:document:sap:rfc:functions">
      <ET_MATERIALS>
        <item>
          <PLANT>3100</PLANT>
          <WAREHOUSE>305R</WAREHOUSE>
          <MATERIAL>500513</MATERIAL>
          <CHARG>ПОКУП</CHARG>
          <QUANTITY>15.000</QUANTITY>
        </item>
      </ET_MATERIALS>
    </n0:ZPM_SOUR_MATERIALSResponse>
  </soap-env:Body>
</soap-env:Envelope>
```

### Пример запроса (ЦиП)

```xml
<soapenv:Body>
  <urn:ZpmSourMaterial>
    <Werks>
      <item>1330</item>
    </Werks>
    <Lgort>
      <item>0014</item>
    </Lgort>
  </urn:ZpmSourMaterial>
</soapenv:Body>
```

### Пример ответа (ЦиП)

```xml
<soap-env:Envelope xmlns:soap-env="http://schemas.xmlsoap.org/soap/envelope/">
  <soap-env:Header/>
  <soap-env:Body>
    <n0:ZpmSourMaterialResponse xmlns:n0="urn:sap-com:document:sap:rfc:functions">
      <EtMaterial>
        <item>
          <Matnr>000000000112011819</Matnr>
          <Werks>1330</Werks>
          <Lgort>0014</Lgort>
          <Charg>ПОКУП</Charg>
          <Labst>2.0</Labst>
        </item>
      </EtMaterial>
    </n0:ZpmSourMaterialResponse>
  </soap-env:Body>
</soap-env:Envelope>
```

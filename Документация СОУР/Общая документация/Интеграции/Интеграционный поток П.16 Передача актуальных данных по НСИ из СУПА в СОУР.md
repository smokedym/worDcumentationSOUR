## Общая информация

- **Направление:** СУПА (SAP) → СОУР.
- **Протокол:** SOAP/XML через ЕИР (Единую интеграционную платформу).
- **Инициатор:** ЕИР/СОУР (периодический запрос по регламенту).
- **Периодичность:** еженедельно, согласно установленному расписанию в 00:00 МСК.

| Параметр      | Значение                                                    |
| ------------- | ----------------------------------------------------------- |
| Направление   | СОУР → СУПА                                                 |
| Протокол      | SOAP/XML                                                    |
| Транспорт     | ЕИР                                                         |
| Триггер       | Периодический запрос по регламенту                          |
| Периодичность | Еженедельно, согласно установленному расписанию в 00:00 МСК |
| Обработка     | Приём и запись в соответствующие справочные таблицы СОУР    |
| Маршрутизация | По HTTP-заголовку `source_platform` (center или cip)        |

## Структура справочников в БД СОУР
Каждый справочник хранится в отдельной таблице. Для обеспечения уникальности используется составное поле:

| Поле БД | Тип данных | Правило формирования |
| :--- | :--- | :--- |
| id | integer | SERIAL PRIMARY KEY (внутренний ID СОУР) |
| source_platform | text | Код ДЗО (center или cip) |
| integration_entity_id | text | ID словаря в мастер-системе (SAP) |
| integration_entity_code | text | FK (внешний ключ) сущности в системе-источнике |
| integration_code | text | Составной ключ: source_platform + & + integration_entity_id + & + integration_entity_code |
| name | text | Наименование на русском языке |

## Ключевые правила обработки

- **Очередность загрузки:** Ввиду зависимостей в первую очередь загружаются независимые справочники: Завод, Виды каталогов, Каталог кодов ТОРО, Группы кодов дефектов, Группа материалов, Единицы измерения.
- **Создание и обновление:** Система проверяет наличие объекта по integration_code. Если объект не найден — он создается; если найден — обновляется.
- **ДЗО:** Определяется по HTTP-заголовку сообщения source_platform.
- **Маски групп дефектов:** В полях ID групп дефектов могут приходить значения с масками (например, DDZ* или *). В этом случае код каталога ТОРО привязывается ко всем группам, начинающимся с указанных символов. Маска * означает привязку ко всем группам.
- **Актуальность оргструктуры:** Перед загрузкой поле active в таблице org_struct очищается. В процессе интеграции записи, пришедшие из СУПА, получают active = true, а отсутствующие в выгрузке помечаются active = false.
- **Группировка измерений:** Данные из потока vid_izm (виды измерений) при записи в таблицу measurement_type должны помечаться типом enum (protocol или statement) согласно предопределенному списку (например, «Индекс состояния» — protocol, «Состояние опоры» — statement).
- **Логика оргструктуры:** Заполнение полей org_struct_id и sup_org_struct_id (вышестоящее подразделение) зависит от наличия в ответе SAP полей FACTORY, PLANNERSGROUP и PRODAREA.

```xml
<soapenv:Body>

    <IV_DATETO></IV_DATETO>

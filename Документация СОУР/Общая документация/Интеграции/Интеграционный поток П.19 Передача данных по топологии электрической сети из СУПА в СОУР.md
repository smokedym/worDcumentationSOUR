## Параметры потока

| Параметр      | Значение                                                                    |
| :------------ | :-------------------------------------------------------------------------- |
| Направление   | СУПА (SAP) → СОУР                                                           |
| Протокол      | SOAP/XML                                                                    |
| Транспорт     | ЕИП (Единая интеграционная платформа)                                       |
| Триггер       | Периодический запрос ЕИП по регламенту или ручной запрос из интерфейса СОУР |
| Периодичность | Еженедельно (автоматически по заводу) / по запросу (вручную)                |
| Обработка     | Запись в таблицу `electric_links`                                           |
| Маршрутизация | По HTTP-заголовку `header_sourcePlatform` (center или cip)                  |

## Общая информация

- **Назначение:** Передача данных о топологии (электрических связях «откуда — куда») между техническими местами для визуализации схем и расчета допустимого радиуса работ на ЛЭП.
- **Инициатор:** ЕИП формирует запрос к СУПА на основании выборки оргструктуры, хранящейся в СОУР.
- **Особенность:** При ручном запросе (кнопка в «Администрировании») СОУР передает ID измененных технических мест, чтобы получить обновление связей только по ним.

## Какие данные передаются

### Запрос (СОУР → СУПА)

| Атрибут запроса (Центр) | Атрибут запроса (ЦиП) | Описание | Источник данных |
| :--- | :--- | :--- | :--- |
| `IV_BEGRU` | `BEGRU` | Код группы полномочий (завод) | `org_struct.integration_entity_code` |
| `IV_DATEFROM` | `DateFrom` | Дата начала периода изменений | Конфигурация сервиса ЕИР |
| `IV_DATETO` | `DateTo` | Дата окончания периода изменений | Конфигурация сервиса ЕИР |
| `TPLNR` | `TPLNR` | Код ТМ для выборочного запроса | `technical_places.integration_entity_id` |

### Ответ (СУПА → СОУР)

| Атрибут XML (Центр) | Атрибут XML (ЦиП) | Поле БД СОУР | Описание |
| :--- | :--- | :--- | :--- |
| `LINKID` | `Kante` | `integration_entity_id` | Код соединения в SAP (строка до 12 симв.) |
| `LINKFROM` | `Tpvon` | `previous_technical_place_id` | ТМ-источник (FK к `technical_places`) |
| `LINKTO` | `Tpnach` | `next_technical_place_id` | ТМ-приемник (FK к `technical_places`) |
| `LINKOBJ` | `Tpkant` | `connect_technical_place_id` | ТМ соединяющего объекта (например, пролет) |
| `NETID` | `Netid` | `type_id` | Тип соединения (FK к `electric_link_connect_types`) |

## Ключевые правила обработки

- **Определение ТМ:** Поля `LINKFROM`, `LINKTO` и `LINKOBJ` сопоставляются с внутренними ID СОУР в таблице `technical_places` по паре `integration_entity_id` + `source_platform`.
- **Тип соединения:** Поле `NETID` (например, E0000001 — схема сетей) сопоставляется со справочником типов соединений.
- **Upsert-логика:** Если связь с таким `LINKID` уже существует, её атрибуты обновляются. Если нет — создается новая запись.
- **Линейные объекты:** Для ТМ, являющихся линиями (ВЛ/КЛ), СОУР использует данные топологии для построения виртуальной «оси» объекта при проверке координат мобильного устройства.
- **Период синхронизации:** Параметры DateFrom/To в запросе позволяют СУПА возвращать только те связи, которые были созданы или изменены за указанное время.

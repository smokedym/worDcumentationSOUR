## Параметры потока

| Параметр      | Значение                                                                        |
| :------------ | :------------------------------------------------------------------------------ |
| Направление   | СУПА (SAP) → СОУР.                                                              |
| Протокол      | SOAP/XML.                                                                       |
| Транспорт     | ЕИР (Единая интеграционная платформа).                                          |
| Триггер       | Ручной запрос пользователя из веб-приложения СОУР.                              |
| Периодичность | По запросу (для каждого РД отдельно).                                           |
| Обработка     | Сохранение файлов в MinIO, запись в таблицы `documents` и `electrical_diagram`. |
| Маршрутизация | По HTTP-заголовку `header_sourcePlatform` (center или cip).                     |

## Общая информация

- **Назначение:** Загрузка схем электроустановок по техническим местам (ТМ), связанным с РД, для их отображения полевым бригадам на мобильных устройствах.
- **Инициатор:** Пользователь веб-приложения СОУР с уровнем доступа к РД «Создание» и выше.
- **Связь с РД:** Загрузка выполняется строго в рамках одного РД. Пользователь выбирает ТМ из списка объектов, на которые выдан данный документ.
- **Особенность:** Если схема в СУПА была изменена, для её обновления в СОУР требуется повторный запрос данных пользователем.

## Запрос (СОУР → СУПА)

> **Примечание:** Атрибуты запроса едины для обоих ДЗО (Центр и ЦиП) без разделения.

| Атрибут запроса | Описание | Источник данных в СОУР |
| :--- | :--- | :--- |
| `TPLNR` | Код технического места в SAP | `technical_places.integration_entity_id`. |
| `DateFrom` | Дата начала периода изменений | Конфигурация сервиса ЕИР. |
| `DateTo` | Дата окончания периода изменений | Конфигурация сервиса ЕИР. |

## Ответ (СУПА → СОУР)

> **Примечание:** Атрибуты ответа едины для обоих ДЗО (Центр и ЦиП) без разделения.

СУПА возвращает асинхронный ответ со схемами по следующим объектам:

1. Запрошенное ТМ верхнего уровня.
2. Все дочерние ТМ от запрошенного.
3. Все единицы оборудования (ЕО), относящиеся к запрошенному ТМ и его дочерним объектам.

| Атрибут XML | Поле БД СОУР | Описание |
| :--- | :--- | :--- |
| `FILE_NAME` | `documents.name` | Наименование файла. |
| `FILE_CONTENT` | — | Содержимое файла (сохраняется в MinIO). |
| `TPLNR` | `electrical_diagram.technical_place_id` | Привязка к техническому месту. |
| `EQUNR` | `electrical_diagram.equipment_id` | Привязка к единице оборудования. |

## Ключевые правила обработки

- **Инициация:** При нажатии кнопки «Загрузить» веб-клиент отправляет mutation-запрос в Ядро с параметрами `rd_id` и списком `technical_place_id`.
- **Очередь запросов:** Ядро создаёт запись в `electrical_diagram_dispatch` со статусом `waiting`. Интеграционный сервис раз в минуту опрашивает эту таблицу и формирует SOAP-сообщение для СУПА.
- **Сохранение и обновление:**
    - Если схема для ТМ уже существует в `electrical_diagram`, файл в MinIO заменяется новым, а ссылка `documents.link` обновляется.
    - Для первого запроса в рамках конкретного РД создается новая запись в `electrical_diagram`, которая может ссылаться на уже существующий `document_id`.
- **Индикация для пользователя:** Пока запрос находится в статусе `waiting` или `in_progress`, в интерфейсе отображается индикатор загрузки. После перехода в статус `done` схемы становятся доступны для просмотра без перезагрузки страницы.
- **Ограничения:**
    - Для объектов, введённых вручную (`custom_object`), загрузка из СУПА недоступна (только ручное прикрепление файлов).
    - Наличие схем не является обязательным условием для смены статуса РД (валидация не накладывается).
- **Очистка данных:** При переходе РД в статусы «Подтверждение работ» или «Отменён» Ядро разрушает связи между `electrical_diagram` и `documents`. Если на конкретный файл в MinIO больше не ссылается ни один РД, запись в `documents` и сам файл удаляются физически.
- **Доступность:** Загрузка и просмотр схем возможны в статусах РД: «Планирование», «Передан на проверку», «Проверен», «Выдан», «В работе». Все загруженные схемы автоматически включаются в пакет синхронизации с мобильным устройством.

**Статусы очереди electrical_diagram_dispatch:**

| Статус | Описание |
|---|---|
| waiting | Запрос создан, ожидает обработки |
| in_progress | SOAP-запрос отправлен в СУПА |
| done | Файл получен и сохранён в MinIO |
| error | Ошибка при получении или сохранении |
## Обработка ошибок

| Ситуация                              | Поведение системы                                               | Действие                                     |
| ------------------------------------- | --------------------------------------------------------------- | -------------------------------------------- |
| СУПА недоступна                       | Retry по расписанию                                             | Автоматически                                |
| TPLNR не найден в СОУР                | Запрос отклоняется                                              | Проверить загрузку объектов через П.17       |
| Файл не получен от СУПА               | Запись в electrical_diagram_dispatch остаётся в статусе waiting | Проверить статус в очереди, повторить запрос |
| MinIO недоступен                      | Файл не сохраняется, ссылка не обновляется                      | Обратиться к администратору                  |
| Объект введён вручную (custom_object) | Загрузка из СУПА недоступна                                     | Прикрепить файл вручную                      |

## Примеры

### Пример запроса

```xml
<soapenv:Body>
  <ZPM_SOUR_DIAGRAM>
    <TPLNR>RP010-0000001</TPLNR>
    <DateFrom>2024-01-01</DateFrom>
    <DateTo>2024-12-31</DateTo>
  </ZPM_SOUR_DIAGRAM>
</soapenv:Body>
```

### Пример ответа

```xml
<soap-env:Envelope xmlns:soap-env="http://schemas.xmlsoap.org/soap/envelope/">
  <soap-env:Header/>
  <soap-env:Body>
    <n0:ZPM_SOUR_DIAGRAMResponse xmlns:n0="urn:sap-com:document:sap:rfc:functions">
      <ET_DIAGRAMS>
        <item>
          <FILE_NAME>Схема_РУ-10кВ.pdf</FILE_NAME>
          <FILE_CONTENT>JVBERi0xLjQKJcOkw7z... (base64)</FILE_CONTENT>
          <TPLNR>RP010-0000001</TPLNR>
          <EQUNR/>
        </item>
      </ET_DIAGRAMS>
    </n0:ZPM_SOUR_DIAGRAMResponse>
  </soap-env:Body>
</soap-env:Envelope>
```

## Параметры потока

| Параметр      | Значение                                                                    |
| :------------ | :-------------------------------------------------------------------------- |
| Направление   | СУПА (SAP) → СОУР                                                           |
| Протокол      | SOAP/XML                                                                    |
| Транспорт     | ЕИП (Единая интеграционная платформа)                                       |
| Триггер       | Периодический запрос ЕИП по регламенту или ручной запрос из интерфейса СОУР |
| Периодичность | Еженедельно (автоматически по заводу) / по запросу (вручную)                |
| Обработка     | Запись в таблицы `technical_places` и `equipments`                          |
| Маршрутизация | По HTTP-заголовку `source_platform` (center или cip)                        |

## Общая информация

- **Назначение:** Передача иерархической структуры технических мест (ТМ) и единиц оборудования (ЕО) из мастер-системы SAP.
- **Инициатор:** ЕИП формирует запрос к СУПА на основании выборки оргструктуры, хранящейся в СОУР.
- **Порядок загрузки:** Сначала загружаются технические места, затем — единицы оборудования.
- **Ограничение:** ЕИП получает данные по всем ЕО, кроме тех, где «тип единиц оборудования» = «T».

## Ключевые правила обработки

### 1. Определение статуса объекта (exploitation_status)

Объект считается активным только при выполнении условий по обоим полям:

- **ЕСЛИ** STATE содержит «МТКУ», **ТО** статус = inactive (false).
- **ЕСЛИ** USERSTATE != «ДСТВ», **ТО** статус = inactive (false).
- **ЕСЛИ** USERSTATE пустой, **ТО** его значение игнорируется и учитывается только поле STATE.
- **ИТОГ:** Если хотя бы один из проверяемых статусов указывает на неактивность, объект помечается как inactive.

### 2. Привязка к организационной структуре

Логика заполнения org_struct_id и sup_org_struct_id на основании ответа SAP:

- FACTORY, PLANNERSGROUP, PRODAREA != null: org_struct_id = Группа плановиков, sup_org_struct_id = Производственный участок.
- FACTORY, PLANNERSGROUP != null, PRODAREA == null: org_struct_id = Группа плановиков, sup_org_struct_id = null.
- FACTORY, PRODAREA != null, PLANNERSGROUP == null: org_struct_id = Производственный участок, sup_org_struct_id = null.
- FACTORY != null, остальные null: org_struct_id = Завод, sup_org_struct_id = null.

### 3. Определение уровня вложенности (depth_level)

Поле depth_level заполняется автоматически на основании количества знаков «-» в integration_entity_id:

- Пример: KS006-0005962 (один дефис) → уровень 1.
- Пример: KS006-0000217-301 (два дефиса) → уровень 2.

### 4. Определение категории объекта (category_id)

Категория технического места определяется автоматически по маске его системного идентификатора (`integration_entity_id`) согласно таблице соответствия:

### Дополнительные системные правила
1.  **Загрузка по заводу:** ЕИП формирует запрос к СУПА на основании выборки оргструктуры из СОУР. Код завода берется из поля `integration_entity_code` таблицы `org_struct` [2].
2.  **Фильтрация по типу «T»:** При получении данных от СУПА единицы оборудования с типом **«T»** игнорируются и в СОУР не загружаются [3].
3.  **Период синхронизации:** Параметры даты и времени создания/изменения объектов (`DateFrom` и `DateTo`) настраиваются в конфигурации сервиса ЕИР [2].
4.  **Очередность обработки:** Процесс интеграции всегда начинается с загрузки (или обновления) технических мест, и только после этого производится добавление или обновление единиц оборудования [4].
5.  **Частота обновления:** Автоматическое обновление данных по заводу выполняется **1 раз в неделю**.

## Параметры потока

| Параметр      | Значение                                                                     |
| :------------ | :--------------------------------------------------------------------------- |
| Направление   | СУПА (SAP) → СОУР.                                                           |
| Протокол      | SOAP/XML.                                                                    |
| Транспорт     | ЕИР (Единая интеграционная платформа).                                       |
| Триггер       | Периодический запрос ЕИР по регламенту или ручной запрос из интерфейса СОУР. |
| Периодичность | Еженедельно (автоматически) / по запросу (вручную).                          |
| Обработка     | Запись в таблицу `measurement_points`.                                       |
| Маршрутизация | По HTTP-заголовку `header_sourcePlatform` (center или cip).                  |

## Общая информация

- **Назначение:** Передача данных о точках измерений (контрольных точках) для технических мест (ТМ) и единиц оборудования (ЕО), необходимых для проведения диагностики и испытаний.
- **Инициатор:** ЕИР формирует запрос к СУПА на основании выборки всех объектов эксплуатации (ТМ и ЕО), хранящихся в СОУР.
- **Связь с признаками (П.18):** Если в записи о точке измерения имеется ссылка на признак (`measurement_attribute_id`), система должна подтягивать значения атрибутов из справочника признаков.

## Какие данные передаются

### Запрос (СОУР → СУПА)

| Атрибут запроса (Центр) | Атрибут запроса (ЦиП) | Описание                         | Источник данных в СОУР                                                           |
| :---------------------- | :-------------------- | :------------------------------- | :------------------------------------------------------------------------------- |
| `EquipmentId` / `TmId`  | `IT_ASSETID`          | Код объекта (ТМ или ЕО) в SAP    | `equipments.integration_entity_id` или `technical_places.integration_entity_id`. |
| `DateBegin`             | `IV_DATEFROM`         | Дата начала периода изменений    | Конфигурация сервиса ЕИР.                                                        |
| `DateEnd`               | `IV_DATETO`           | Дата окончания периода изменений | Конфигурация сервиса ЕИР.                                                        |
|                         |                       |                                  |                                                                                  |

### Ответ (СУПА → СОУР)

| Атрибут XML (Центр)    | Атрибут XML (ЦиП) | Поле БД СОУР                          | Тип данных в БД | Описание                        |
| :--------------------- | :---------------- | :------------------------------------ | :-------------- | :------------------------------ |
| `EquipmentId` / `TmId` | `ASSETID`         | `technical_place_id` / `equipment_id` | integer         | Внутренний ID объекта в СОУР.   |
| `ParamId`              | `PRIZNAK`         | `measurement_attributes_id`           | integer         | Ссылка на справочник признаков. |
| `PointId`              | `POINT`           | `integration_entity_id`               | text            | Код точки измерения в SAP.      |
| `PointName`            | `POINTNAME`       | `name`                                | text            | Наименование точки.             |
| `PointPos`             | `POINTPARAM`      | `measurement_position`                | text            | Позиция измерения.              |
| `PointDateCreate`      | `CREATEDATE`      | `created_at`                          | timestamp       | Дата и время создания точки.    |

## Ключевые правила обработки

- **Идентификация объектов:** По коду `ASSETID` система определяет принадлежность точки к таблице `technical_places` или `equipments`, используя фильтр по `source_platform`.
- **Синхронизация с признаками:** При вставке точки измерения, если указан `PRIZNAK` (или `ParamId`), система проверяет наличие соответствующего признака в таблице `measurement_attribute`.
- **Копирование значений:** Из таблицы признаков (обновляемой потоком П.18) в запись точки измерения (`measurement_points`) копируются следующие поля: `value_char`, `value_from`, `value_to`, `measurement_unit_id`.
- **Фильтрация:** ЕИР получает данные по всем объектам, кроме единиц оборудования с типом «T».
- **Обработка сбоев:** При недоступности системы СУПА или сбоях шины инициируется логика Retry (повторная отправка).
  
## Обработка ошибок

| Ситуация                                  | Поведение системы                       | Действие                                  |
| ----------------------------------------- | --------------------------------------- | ----------------------------------------- |
| СУПА недоступна                           | Retry по расписанию                     | Автоматически, вмешательство не требуется |
| ASSETID не найден в СОУР                  | Запись пропускается, логируется         | Проверить наличие объекта в П.17          |
| PRIZNAK не найден в measurement_attribute | Точка создаётся без значений атрибутов  | Дождаться обновления П.18                 |
| Дубль PointId для одного объекта          | Обновление существующей записи (upsert) | —                                         |
## Примеры

### Пример запроса (Центр)

```xml
<soapenv:Body>
  <ZPM_SOUR_MPOINT>
    <EquipmentId>PS035-000345</EquipmentId>
    <DateBegin>2024-03-18</DateBegin>
    <DateEnd>2024-03-21</DateEnd>
  </ZPM_SOUR_MPOINT>
</soapenv:Body>
```

</soap-env:Envelope>

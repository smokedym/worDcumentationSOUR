
## Общая информация

- **Направление:** СУПА (SAP) → СОУР
- **Протокол:** SOAP/XML через ЕИР (Единую интеграционную платформу)
- **Инициатор:** СУПА (транзакция ZPM_ORD_SOURSENT)
- **Периодичность:** по запросу (ручная или автоматическая отправка из SAP)

## Какие данные передаются

- Заказы ТОРО (плановые и аварийные)
- Операции заказа (одна или несколько)
- Физические объёмы (привязаны к операциям)
- Плановые материалы (привязаны к операциям)
- Технологические карты (для комплексных заказов)

## Структура заказа

1. Данные заказа
2. Операции заказа
   - Данные операций
   - Физические объёмы
   - Материалы
3. Технологические карты

## Виды заказов

- **Комплексный (долгосрочный) заказ:** Заказ на комплексный объект. Для ДЗО «Центр» определяется по виду работ (KK, KR, RS) и маске технического места. Для ДЗО «ЦиП» определяется по типу заказа (PMKO, PMKT) и содержит ID подчинённых заказов.
- **Подчинённый заказ:** Заказ, привязанный к комплексному. Содержит операции, материалы и физические объёмы.

## Ключевые правила обработки

- **Определение ДЗО:** По HTTP-заголовку `sourse_platform` (center или cip).
- **Техническая операция:** Операция с IDKO = 'SPOSOB' (способ выполнения) не записывается в БД и игнорируется.
- **Ключ образца (IDKO):** Обязателен для всех операций, кроме заказов определённых типов (PMKO, PMPR, PMNR, PMKT, PMTO для ЦиП). Если ключ образца пустой — заказ отклоняется.
- **Отзыв заказа:** При ACTIVITY = 3 заказ удаляется из СОУР, а связанные подчинённые заказы открепляются.

## Коды ответов (из SAP)

- 001: Заказ уже существует
- 005: Заказ не существует
- 006: Id справочника для TaskType отсутствует
- 007: Id справочника для TaskReason отсутствует
- 010: Id справочника для WorkOperation отсутствует
- 101: Заказ успешно создан
- 102: Заказ успешно обновлён
- 103: Заказ успешно отозван

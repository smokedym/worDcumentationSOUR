## Параметры потока

| Параметр      | Значение                                                                     |
| :------------ | :--------------------------------------------------------------------------- |
| Направление   | СУПА (SAP) → СОУР.                                                           |
| Протокол      | SOAP/XML.                                                                    |
| Транспорт     | ЕИР (Единая интеграционная платформа).                                       |
| Триггер       | Периодический запрос ЕИР по регламенту или ручной запрос из интерфейса СОУР. |
| Периодичность | Еженедельно (автоматически) / по запросу (вручную).                          |
| Обработка     | Запись в таблицу `detected_defects`.                                         |
| Маршрутизация | По HTTP-заголовку `header_sourcePlatform` (center или cip).                  |

## Общая информация

- **Назначение:** Синхронизация данных о дефектах, зафиксированных в мастер-системе SAP, для их отображения в «Листках осмотра» на мобильных устройствах и учета при планировании работ.
- **Инициатор:** Интеграционная шина (ЕИР). Раз в неделю формируется запрос к СУПА на основании выборки всех ТМ и ЕО, хранящихся в СОУР.
- **Особенность:** Поток также используется для идентификации «Опасных мест» для сегмента ЦиП на основании специфических кодов дефектов.

## Какие данные передаются

### Запрос (СОУР → СУПА)

| Атрибут (Центр)        | Атрибут (ЦиП)   | Описание                | Источник данных в СОУР   |
| :--------------------- | :-------------- | :---------------------- | :----------------------- |
| ``EquipmentId / TmId`` | ``IT_ASSETID``  | Код объекта (ТМ или ЕО) | integration_entity_id    |
| ``DateBegin``          | ``IV_DATEFROM`` | Дата начала периода     | Конфигурация сервиса ЕИР |
| ``DateEnd``            | ``IV_DATETO``   | Дата окончания периода  | Конфигурация сервиса ЕИР |
|                        |                 |                         |                          |

### Ответ (СУПА → СОУР)

| Атрибут XML (Центр) | Атрибут XML (ЦиП) | Поле БД СОУР | Тип данных | Описание |
| :--- | :--- | :--- | :--- | :--- |
| `ASSETID` | `tplnr` / `equnr` | `technical_place_id` / `equipment_id` | integer | Внутренний ID объекта в СОУР |
| `DEFECTTYPE` | `fecod` | `defect_code_id` | integer | Ссылка на справочник кодов дефектов |
| `DEFECTTYPEGROUP` | `fegrp` | `defect_code_id` → `group_id` | integer | Ссылка на справочник групп дефектов |
| `DEFECTSTATUS` | `systemState` | — | text | Статус дефекта (влияет на флаг `confirmed`) |
| `DEFECTNUM` | — | `integration_entity_id` | text | Номер дефекта в SAP |
| `INSPECTIONDATETIME` | — | `record_at` | timestamp | Дата и время обнаружения |
| `ENGINEER` | — | `employee_id` | integer | Табельный номер обнаружившего |
| `COMM` | `qmtxt` | `comment` | text | Текстовое описание |
| — | — | `id_export_defect` | integer | Внутренний ID, присваиваемый СОУР |

## Ключевые правила обработки

### 1. Статусная модель дефектов
Система СОУР обрабатывает входящие дефекты в зависимости от их статуса в СУПА:

- **Записываются как активные (confirmed = true):**
    - СООТ — дефект выявлен, но еще не включен в заказ ТОРО.
    - СОРА — дефект выявлен и включен в заказ ТОРО.
    - СОРА + ЗКПР — дефект включен в заказ, который находится в процессе выполнения.
- **Записываются как неподтвержденные (confirmed = false), но остаются в БД:**
    - СОЗА — сообщение о дефекте закрыто.
    - Статус содержит маску `%МТКУ%` (метка удаления). Запись не удаляется, но помечается как неподтверждённая.
- ** Физически удаляются из СОУР:**
    - Если получен статус МТКУ (без маски, точное совпадение), существующая запись о дефекте полностью удаляется из таблицы `detected_defects`.

### 2. Логика «Опасных мест» (только для ЦиП)
Если в потоке П.24 приходит дефект с определенным кодом, объект автоматически помечается как Опасное место в таблице `dangerous_places`:

- Код 0652 — «Подъем запрещен (нет в ИС)».
- Код 0871 — «Шунт поврежденного разъединителя».
- **Важно:** Информация, полученная через дефекты (П.24), имеет приоритет над информацией из признаков (П.18) и не может быть перезаписана ими.

### 3. Сопоставление и валидация

- **Объекты:** Поиск ТМ или ЕО идет по паре `integration_entity_id` + `source_platform`.
- **Справочники:** Если коды `fecod` или `fegrp` отсутствуют в локальных справочниках НСИ (поток П.16), запись дефекта отклоняется с ошибкой.
- **Ошибки:** При недоступности СУПА или сбое ЕИР используется механизм Retry.


## Обработка ошибок

| Ситуация                            | Поведение системы                       | Действие                               |
| ----------------------------------- | --------------------------------------- | -------------------------------------- |
| СУПА недоступна                     | Retry по расписанию                     | Автоматически                          |
| ASSETID не найден в СОУР            | Запись пропускается, логируется         | Проверить загрузку объектов через П.17 |
| fecod / fegrp нет в справочнике НСИ | Запись дефекта отклоняется              | Проверить актуальность П.16            |
| Получен статус МТКУ                 | Дефект удаляется из БД                  | Автоматически                          |
| Дубль DEFECTNUM для одного объекта  | Обновление существующей записи (upsert) | —                                      |
## ## Примеры

### Пример запроса (Центр)

```xml
<soapenv:Body>
  <ZPM_SOUR_DEFECT_EXP>
    <EquipmentId>PS110-000344-01</EquipmentId>
    <DateBegin>2024-03-18</DateBegin>
    <DateEnd>2024-03-21</DateEnd>
  </ZPM_SOUR_DEFECT_EXP>
</soapenv:Body>
```

### Пример ответа (Центр)

```xml
<soap-env:Envelope xmlns:soap-env="http://schemas.xmlsoap.org/soap/envelope/">
  <soap-env:Header/>
  <soap-env:Body>
    <n0:ZPM_SOUR_DEFECT_EXPResponse xmlns:n0="urn:sap-com:document:sap:rfc:functions">
      <ET_DEFECTS>
        <item>
          <ASSETID>PS110-000344-01</ASSETID>
          <DEFECTTYPE>0001</DEFECTTYPE>
          <DEFECTTYPEGROUP>DK000049</DEFECTTYPEGROUP>
          <DEFECTSTATUS>СООТ</DEFECTSTATUS>
          <DEFECTNUM>010000005889</DEFECTNUM>
          <COMM>Осмотр: Трансформатор ТДТН-25000/110/35</COMM>
        </item>
      </ET_DEFECTS>
    </n0:ZPM_SOUR_DEFECT_EXPResponse>
  </soap-env:Body>
</soap-env:Envelope>
```

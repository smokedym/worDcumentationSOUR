Общая информация

| Параметр      | Значение                                                    |
| ------------- | ----------------------------------------------------------- |
| Направление   | СОУР → СУПА                                                 |
| Протокол      | SOAP/XML                                                    |
| Транспорт     | ЕИР                                                         |
| Триггер       | Нажатие кнопки «Передать в СУПА»                            |
| Обработка     | Таблица-очередь toro_order_dispatch, задание каждые 5 минут |
| Маршрутизация | По заголовку header_sourcePlatform (center / cip)           |

Какие данные передаются

- Фактические даты начала и окончания работ по заказу.
- Фактические трудозатраты (человеко-часы) в разрезе сотрудников и рабочих мест.
- Фактические физические объёмы по каждой операции.
- Фактически использованные материалы (номенклатура, количество, склад списания).
- Фактические транспортные затраты (время перемещения).

Структура передаваемых данных

Сообщение формируется отдельно для каждого заказа ТОРО (включая подчинённые блоки для комплексных заказов):

Часть заказа ТОРО (IT_FACTORDER.item)

|Тег сообщения|Источник данных (БД СОУР)|Примечание|
|---|---|---|
|`ID`|`toro_orders.integration_entity_id`|Номер заказа в SAP.|
|`FACTSTART`|`toro_orders.fact_start`|Фактическое начало работ.|
|`FACTFINISH`|`toro_orders.fact_end`|Фактическое окончание работ.|

Часть операций (OPERATION.item)

|Тег сообщения|Источник данных (БД СОУР)|Примечание|
|---|---|---|
|`VORNR_OPER`|`toro_operations.integration_entity_id`|Идентификатор операции.|
|`FACTWORKHOURS`|`toro_operations.fact_duration_in_human_hours`|Фактические Ч/Ч по операции.|

Часть физических объёмов (FACTFO.item)

|Тег сообщения|Источник данных (БД СОУР)|Примечание|
|---|---|---|
|`FOIDFACT`|`toro_operation_physical_volumes.integration_entity_id`|Код физического объема.|
|`QUANTITYFACT`|`toro_operation_physical_volumes.fact`|Фактическое количество.|

Часть материалов (COMPONENT.item)

|Тег сообщения|Источник данных (БД СОУР)|Примечание|
|---|---|---|
|`VORNR_COMP`|`toro_operations.integration_entity_id`|Операция, к которой привязан материал.|
|`MATNR`|`materials.integration_entity_id`|Код материала в SAP.|
|`FACT`|`material_on_toro_operation_fact.count`|Фактическое количество списанного МТР.|

Дополнительно: Блок комплексного заказа

Если заказ привязан к комплексному (КО), формируется блок данных для КО:

- `ID`: `complex_toro_orders.integration_entity_id`.
- `FACTWORKHOURS`: `toro_orders.duration_transport_in_human_hours` (затраты на перемещение к объекту).
- `VORNR_OPER`: `complex_toro_orders.toro_operation_entity_id`.

Ключевые правила обработки

- **Критическое условие:** Для успешной передачи факта в операциях заказа обязательно должен быть заполнен **«ключ образца» (KTSCH / sample_key_id)**. Если его нет в SAP, данные из СОУР не будут приняты.
- **Алгоритм отправки подчинённых заказов:** После отправки части подчинённых заказов в SAP остальные блокируются в СОУР до получения подтверждения.
- **Распределение времени:** Время нахождения РД в статусе «В работе» распределяется между членами бригады. Время, не отнесенное к «В работе», считается временем доставки и относится на отдельный заказ ТОРО.
- **ДЗО:** Маршрутизация сообщения в шине идет по заголовку `header_sourcePlatform` (`center` или `cip`).
- **Формат дат:** Все даты передаются строго в UTC с указанием часового пояса (например, `2025-02-17T08:00:00.00Z`).

Коды ответов (от СУПА)

| Код | Значение | Поведение СОУР | Действие пользователя |
|---|---|---|---|
| 200 | Успех | РД переходит в статус «Закрыт» | — |
| 404 | Ошибка данных | Автоповтор отправки | Если повтор не помог — проверить ключ образца в SAP |
| Warning | Заказ уже в работе | Логируется | Обратиться в техподдержку |
